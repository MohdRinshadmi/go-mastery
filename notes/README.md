# Notes — Quick Reference for Elite GoLang Mastery

> Mentor note: The day-by-day lessons teach you *why*. These notes are the *recall layer* — the stuff you should be able to produce from memory in a PR review or an interview. Keep them open in a split pane while you work. They are deliberately dense and skimmable, not tutorial-paced.

## Files

| File | What it's for |
|------|---------------|
| [cheatsheet.md](cheatsheet.md) | Dense Go syntax + stdlib + `go` CLI reference. Code-snippet heavy. Reach for it when you've forgotten "how do I write X again?" |
| [glossary.md](glossary.md) | ~50 terms every Go engineer must define on demand, one tight sentence each. Alphabetized. |
| [pitfalls.md](pitfalls.md) | The consolidated gotchas list, grouped by topic. Each entry: the trap, why it bites, the fix. These are the bugs that ship to prod. |

## How to use these as a quick reference / spaced-repetition aid

Treat the cheatsheet as a **lookup table** (open it, find the snippet, copy the shape, close it) and treat the glossary + pitfalls as **flashcards**. The highest-leverage drill: cover the right-hand column and try to produce it. For the glossary, read the term and say the one-sentence definition out loud before reading it. For pitfalls, read the "trap" line and try to state *why it bites* and *the fix* before scrolling. Do this in short passes — five minutes at the start of each day, re-testing yesterday's day plus one older day on a [spaced schedule](https://en.wikipedia.org/wiki/Spaced_repetition) (day 1, day 2, day 4, day 8...). The items you miss are exactly the ones to bring to your mentor. The goal isn't to re-read until it "looks familiar" — familiarity is a liar. The goal is **active recall under a blank page**, because that's the condition you'll be in during a real review or a 2 a.m. incident.
